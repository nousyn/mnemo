# Dev Log 009: v1.1.0 混合搜索 + 渐进式展示

## 日期

2026-03-05

## 概述

v1.1.0 实现了两个新特性：**混合搜索（Hybrid Search）** 和 **渐进式展示（Progressive Disclosure）**。

## 特性 1: 混合搜索

### 设计

在原有向量搜索基础上增加关键词检索路径，两者结果合并排序。

- **关键词搜索**: 将查询按空格拆分为词项，对每条笔记内容做大小写不敏感的子串匹配，得分 = 匹配词项数 / 总词项数
- **结果合并**: 向量权重 0.7，关键词权重 0.3；两种路径都找到的笔记合并分数，仅一种找到的保留加权分数
- **降级策略**: embedding 模型未就绪时，仅使用关键词搜索

### 实现

修改 `src/core/embedding.ts`:

- 新增 `SearchResult` 接口导出
- 新增 `keywordSearch()` 内部函数
- 新增 `mergeResults()` 内部函数
- 重写 `searchNotes()` 为混合版本

**外部 API 不变** — `searchNotes(query, topK, sourceFilter)` 签名保持不变。无新依赖。

## 特性 2: 渐进式展示

### 设计

`memory_search` 不再返回完整内容，改为返回摘要 + ID + 元数据。新增 `memory_get` 工具按 ID 获取完整内容。

### 实现

1. **修改 `src/tools/search.ts`**:
   - 移除加载完整笔记的逻辑（原来通过 `readNote` 获取全文）
   - 移除 `isEmbeddingReady()` 前置检查（混合搜索自身处理降级）
   - 新增 `extractSummary()` 导出函数：取第一行或前 200 字符
   - 输出格式改为 `Summary:` 字段
   - 提示用户使用 `memory_get` 获取完整内容

2. **新建 `src/tools/get.ts`**:
   - `memory_get` 工具：接受 `ids` 数组，返回完整笔记内容
   - 处理部分 ID 不存在的情况（返回找到的 + 未找到列表）

3. **修改 `src/index.ts`**:
   - 导入并注册 `registerGetTool`
   - 工具总数从 6 增至 7

4. **修改 `src/prompts/templates.ts`**:
   - MEMORY_PROMPT 新增 "When to get full memory content (memory_get)" 段落
   - 更新 memory_search 说明，提示返回摘要

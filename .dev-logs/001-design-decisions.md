# 001 - 设计讨论纪要

**日期：** 2026-03-05

---

## 项目概述

**项目名称：** Mnemo（取自希腊记忆女神 Mnemosyne）

**核心问题：** Agent 对话中有价值的信息会随上下文截断而丢失

**解决方式：** 在对话过程中自动蒸馏关键信息为便签，跨会话持久化，按需召回

---

## 设计灵感

借鉴黑板教学场景：老师在黑板上写解题思路，写满后擦掉部分内容，但擦掉前将关键内容提纯精炼记到小纸条上，学生回顾时仍可参考。

对应技术映射：

| 比喻     | 技术对应                        |
| -------- | ------------------------------- |
| 大黑板   | LLM 的 context window           |
| 写满了   | token 上限                      |
| 擦掉内容 | 滚动窗口裁剪历史                |
| 提纯精炼 | LLM 蒸馏压缩                    |
| 小纸条   | 持久化记忆便签（Markdown 文件） |

---

## 技术决策

### 已确认

| 决策项         | 结论                                      | 原因                                     |
| -------------- | ----------------------------------------- | ---------------------------------------- |
| 实现方式       | MCP 服务（stdio 模式）                    | 架构标准化、复用性好、按需启动无常驻负担 |
| 开发语言       | TypeScript                                | MCP SDK 支持最成熟                       |
| 记忆存储       | 本地 Markdown 文件                        | 人类可读、LLM 友好、持久化简单           |
| 语义检索       | 本地 embedding（vectra）                  | 离线可用、延迟低、纯 TS 无外部依赖       |
| 压缩蒸馏       | LLM 自行蒸馏，不单独做重要度打分          | 蒸馏过程自带筛选判断                     |
| 压缩触发       | 提示词引导 + memory_save 时程序兜底       | 不完全依赖 LLM 自觉                      |
| 记忆隔离       | 所有记忆存同一空间，靠语义检索 + 标签过滤 | 不按 session/目录做物理隔离              |
| 多工具共享     | 不同 Agent 工具共享同一份记忆             | 记忆属于用户，不属于工具                 |
| 首个适配       | OpenCode                                  | 原生支持 MCP，日常使用                   |
| Agent 类型识别 | MCP 配置枚举指定 + 运行时检测             | 双重保障                                 |

### 关键设计

**记忆便签元数据字段（最小集）：**

```yaml
id: string
created: timestamp
updated: timestamp
tags: string[]
source: string # 来自哪个 agent 工具
```

**提示词注入：** 通过 memory_setup tool 自动写入 Agent 配置文件（如 OPENCODE.md），tool 自己完成文件操作，不依赖 LLM 写入。

**数据目录结构：**

```
memory/
  notes/
```

扁平结构，未来需要隔离时再加层级。

---

## MCP 工具集（第一版）

```
memory_setup(agent_type?)   -> 初始化，写入提示词到配置文件
memory_save(content, tags?) -> 保存记忆便签
memory_search(query)        -> 语义检索相关记忆
memory_compress()           -> 蒸馏压缩
```

---

## 第一版不做

- 多用户隔离
- 多 Agent 共享记忆
- 便签合并 / 归档策略
- 适配其他 Agent 工具（Claude Code / Openclaw / Codex）
- memory_list 高级筛选

---

## 未来扩展方向

- 多工具隔离：在检索时按 source 标签过滤即可，无需改存储结构
- 多用户：在 user 层级加目录隔离
- 共享记忆：引入"记忆管理员"角色统一写入

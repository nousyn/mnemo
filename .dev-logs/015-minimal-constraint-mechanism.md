# 015 - Mnemo 最小约束机制

## 背景

在通用层 v2 与开发领域层 v1 确认后，Mnemo 已经具备了“记什么”的初步分类模型。

但如果只有分类，没有约束，Mnemo 仍然会快速滑向以下问题：

- 什么都记一点，最后变成噪音堆积
- 同一条信息被重复保存，越积越乱
- 讨论过程被误记为稳定结论
- 已经失效或已闭环的上下文继续长期残留

因此，在设计触发机制之前，必须先建立一套最小约束机制。

这里的目标不是构建复杂治理系统，而是先建立最基础、最必要的控制规则，让 Mnemo 保持“高价值长期上下文系统”的产品定位。

## 约束目标

最小约束机制要解决 4 个核心问题：

1. 防止把短期噪音写进长期记忆
2. 防止同类信息重复堆积
3. 防止过程性信息被误当作事实或规则
4. 防止已经闭环的信息长期悬挂不清

## 约束一：保存阈值

Mnemo 不是全量记录系统，因此每条记忆都必须先满足保存阈值。

### 进入 Mnemo 的基本判断

一条信息至少满足以下条件中的 2 条，才应考虑进入 Mnemo：

- 跨会话仍然有用
- 会影响未来的协作、判断或执行方式
- 不记下来，下次大概率需要重新对齐或重复踩坑

如果不满足这 3 条中的 2 条，则默认不进入 Mnemo。

### 默认不保存的内容

以下内容默认不进入 Mnemo：

- 临时任务状态
- 普通命令输出
- 一次性调试细节
- 未确认的猜测
- 很快会过期的小信息
- 仅在当前轮对话有价值的碎片上下文

## 约束二：分类边界

同一条信息在保存时必须先判断其所属类型，避免把不同语义混写在一起。

### 核心分类判断

- `fact`：客观稳定信息，回答“是什么”
- `decision`：已拍板结果，回答“这次定什么”
- `rule`：长期约定，回答“以后通常怎么做”
- `goal`：长期方向
- `continuity`：待续线头

### 分类优先级

当一条信息可能落入多个类别时，按以下优先级判断：

1. 是否是纯客观稳定信息 -> `fact`
2. 是否是本轮刚拍板的结果 -> `decision`
3. 是否已经形成长期默认做法 -> `rule`
4. 是否是长期方向 -> `goal`
5. 是否是尚未闭环的线头 -> `continuity`

目的不是绝对排他，而是先给 Mnemo 一个稳定的默认落点。

## 约束三：演化与沉淀

Mnemo 的分类不是静止的，记忆会随着时间发生演化。

### 允许的演化路径

- `continuity -> decision`
- `decision -> rule`
- `continuity -> fact`
- `experience -> rule`

例如：

- 一个尚未定案的问题，定下来后可转为 `decision`
- 一个被多次复用的决策，可沉淀为 `rule`
- 一个长期验证有效的经验，可沉淀为 `rule`

### 不建议的情况

- 未闭环内容直接当 `fact`
- 一次性选择直接当 `rule`
- 单次报错直接上升为 `experience`

## 约束四：去重与收敛

Mnemo 不应默认不断新增同类记忆，而应优先收敛。

### 同类信息的处理顺序

当出现相似内容时，优先顺序应为：

1. 是否补充已有记忆
2. 是否更新已有记忆
3. 是否替换过时记忆
4. 最后才是新增记忆

### 典型例子

- 同一用户偏好出现更清晰表述 -> 更新原 `preference`
- 同一开放事项有新进展 -> 更新原 `continuity`
- 决策已沉淀成长期规则 -> 新增或转化为 `rule`，并弱化原 `decision`

## 约束五：闭环机制

Mnemo 中不是所有记忆都应永久保持同样强度。

尤其是 `continuity` 类内容，如果已经闭环，就不应继续以“待续线头”的形式存在。

### `continuity` 的处理规则

- 未完成时保留为 `continuity`
- 闭环后：
  - 若形成结论 -> 转为 `decision`
  - 若形成稳定事实 -> 转为 `fact`
  - 若形成长期约定 -> 转为 `rule`
  - 若已失去长期价值 -> 直接淡出，不再保留

这保证 Mnemo 不会被大量陈旧线头污染。

## 约束六：经验进入门槛

`experience` 是最容易膨胀的类别，因此必须额外提高进入门槛。

### `experience` 只记录这类内容

- 已被验证过
- 可复用
- 会长期影响后续协作或判断

### `experience` 默认不记录这类内容

- 单次错误
- 单次报错全文
- 尚未确认有效的处理方法
- 只在当前任务有效的小技巧

也就是说：

- 错误本身不是经验
- 经验证可复用的处理方式，才可能成为经验

## 最小约束机制的产品意义

这套机制的意义不是增加复杂度，而是建立最基础的质量底线：

- Mnemo 记的是高价值长期上下文，而不是短期噪音
- Mnemo 追求收敛，而不是无限累积
- Mnemo 允许记忆演化，而不是静态堆积

这也是后续触发机制能够成立的前提。否则触发做得越好，只会越快把噪音写进系统。

## 结果

Mnemo 最小约束机制当前确认了 6 条基础规则：

1. 保存阈值
2. 分类边界
3. 演化与沉淀
4. 去重与收敛
5. 闭环机制
6. 经验进入门槛

后续触发机制设计将以这 6 条约束作为底层规则，不再直接从“什么时候方便触发”出发，而是从“什么内容值得写入长期上下文”出发。

# 013 - Mnemo 通用层记忆模型 v2（可用版）

## 背景

Mnemo 新阶段已经确认：

> Mnemo 不是全量记忆系统，而是高价值长期上下文系统。

在这个前提下，首先需要解决的不是触发机制，而是先定义清楚：

- Mnemo 到底应该记什么
- 哪些内容属于长期上下文
- 哪些内容不应该进入 Mnemo

因此，本轮先收敛一版面向所有 agent 用户都适用的通用层记忆模型。

## 通用层目标

通用层不是面向开发者专属，而是面向所有使用 agent 的人：

- 开发者
- 产品/创业者
- 写作者
- 研究者
- 运营/分析人员
- 个人助理型用户

所以它必须抽象到足够通用，但又不能泛化到“什么都能装”。

## 通用层 v2 分类

### 1. `preference`

用户偏好与协作习惯。

适合记录：

- 表达风格偏好
- 工作方式偏好
- 决策偏好
- 禁忌与边界

例子：

- 先讨论方案再实现
- 输出用中文
- 不要过度设计

### 2. `profile`

用户、项目或长期主题的稳定背景信息。

适合记录：

- 身份角色
- 长期背景
- 常用工具/环境
- 稳定的主体信息

例子：

- 用户长期在做 AI 记忆产品
- 常用 OpenCode / OpenClaw
- 某项目长期使用 TypeScript

### 3. `goal`

长期想达成的方向、目标、主题。

适合记录：

- 长期产品方向
- 持续推进的主题
- 阶段性但非短期的目标

例子：

- 把 Mnemo 做成高价值长期上下文系统
- 推进 metis 项目

边界：

- `goal` 记录的是方向，不是待办
- 一次任务完成后仍然成立的内容，通常属于 `goal`

### 4. `continuity`

当前尚未闭环，但未来必须继续接上的上下文线索。

这是此前工作名 `open_loop` 的正式收敛结果。

适合记录：

- 尚未定案但已开启的关键问题
- 后续一定会继续推进的未闭环事项
- 下次对话必须接续的上下文状态

例子：

- 通用层分类尚未完全收敛
- metis 的边界和 MVP 还未定案
- Mnemo 触发机制方案尚未落地

边界：

- `continuity` 是待续的线头，不是长期愿景
- 一旦闭环，就应关闭、移除，或演化为 `decision` / `rule` / `fact`

### 5. `fact`

稳定、客观、会被反复用到的信息。

适合记录：

- 客观配置
- 环境事实
- 项目事实
- 路径、命名、资源定位等

例子：

- 包名
- 仓库地址
- 分支策略
- 支持哪些 agent 类型

边界：

- `fact` 回答的是“是什么”
- 不表达选择过程，也不表达协作约定

### 6. `decision`

已经拍板的选择与共识。

适合记录：

- 已确认方案
- 已排除方向后的结果
- 对后续有持续影响的明确决策

例子：

- `memory_setup` 默认值改成 `global`
- 先做通用层/领域层，再做触发机制
- 新项目命名为 `metis`

边界：

- `decision` 回答的是“这次定什么”
- 它通常是一次拍板结果，而不是通用约定

### 7. `rule`

可反复适用的协作规则、工作约定、默认做法。

适合记录：

- 输出约定
- 工作流约定
- 长期协作协议

例子：

- 推荐项优先说
- roadmap 记录长期方向，编号日志记录阶段留档
- 编号日志可按阶段灵活分段记录

边界：

- `rule` 回答的是“以后通常怎么做”
- `decision` 经过多次验证后，可能沉淀为 `rule`

### 8. `experience`

被验证过、可复用的高价值经验。

适合记录：

- 长期有价值的方法
- 经验证有效的做法
- 反复有用的经验教训

例子：

- 某种工作方式更适合当前用户
- 某类问题处理时应优先采用的做法

边界：

- `experience` 不是所有错误日志的收容器
- 只有已经证明会长期影响后续协作的经验，才适合进入 Mnemo

## 分类边界总结

### `goal` vs `continuity`

- `goal`：长期方向
- `continuity`：待续的具体线头

判断方法：

- 完成后仍然成立 -> `goal`
- 完成后应关闭或转化 -> `continuity`

### `fact` vs `decision` vs `rule`

- `fact`：客观稳定信息，回答“是什么”
- `decision`：一次拍板结果，回答“这次定什么”
- `rule`：反复适用约定，回答“以后通常怎么做”

演化关系：

- `decision` 经过多次验证后，可以沉淀为 `rule`
- `continuity` 闭环后，可能转化为 `decision`、`rule` 或 `fact`

## 暂不纳入通用层的内容

以下内容暂不作为通用层一级分类：

- `task`：过于短期，更像执行态信息
- `error`：更适合作为经验来源，后续更偏向 metis 处理
- `constraint`：当前先并入 `rule` 或 `fact`
- `context`：过于宽泛，容易沦为兜底垃圾桶

## 使用原则

这版通用层模型的核心不是“分类越多越好”，而是：

- 让 Mnemo 只存高价值长期上下文
- 让后续触发机制有明确语义目标
- 让未来的结构化、去重、压缩、演化有基础可依

## 结果

通用层 v2 可用版最终保留 8 个一级类别：

- `preference`
- `profile`
- `goal`
- `continuity`
- `fact`
- `decision`
- `rule`
- `experience`

这版先作为后续领域层与触发机制设计的基础模型使用。
